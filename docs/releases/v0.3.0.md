# OhMyCodex v0.3.0

v0.3.0 is a breaking release. Every public Skill moves to the `omc-*` namespace, with no compatibility aliases.

## Migration

| Before v0.3 | v0.3 |
| --- | --- |
| `ohmycodex-orchestrator` | `omc-orchestrator` |
| `ohmycodex-init` | `omc-init` |

Update saved prompts and start a new task after reinstalling. Portable invocation is `$omc-*` or `/skills`; `/omc-*` resolution is a client convenience, not a custom command parser.

## New capabilities

- `omc-doctor` reports active controls and safe degradation routes.
- `omc-intentgate`, `omc-loop`, and `omc-letgo` add native Goal-backed continuation with an evidence-backed configurable blocker threshold and no separate total iteration limit.
- Scheduled heartbeat is limited to external waits and cleaned on terminal outcomes.
- `omc-cn` and `omc-en` transactionally switch plugin-owned English/Simplified Chinese metadata. Restart Codex or start a new task after switching.
- The Capability Resolver, Loop Ledger, and Locale Manager use only the Python standard library.

## Deliberate exclusions

This release adds no custom runtime, shell loop, Stop Hook, daemon, scheduler, MCP server, LSP server, AST server, app, model provider, or telemetry. Codex-native permissions, trust, authentication, Goal, Scheduled, MCP, release, push, deploy, tag, and publication controls remain authoritative.
