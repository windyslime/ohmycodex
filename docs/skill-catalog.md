# Skill catalog

All skills are instruction-only and remain useful when a host lacks shell, file-write, browser, or Git access. Their shared rule is simple: inspect evidence first, preserve user control, and record verified outcomes when `.ohmycodex/` is writable.

| Skill | Primary output |
| --- | --- |
| `ohmycodex-orchestrator` | Route and next decision |
| `ohmycodex-init` | Project profile and workspace |
| `ohmycodex-discover` | Discovery brief |
| `ohmycodex-spec` | Approved MVP specification |
| `ohmycodex-architecture` | Architecture and change plan |
| `ohmycodex-implement` | Implemented milestone evidence |
| `ohmycodex-qa` | Verification evidence |
| `ohmycodex-debug` | Root-cause record and regression coverage |
| `ohmycodex-refactor` | Behavior-preserving refactor record |
| `ohmycodex-review` | Review findings and recommendation |
| `ohmycodex-release` | Release checklist and go/no-go |
| `ohmycodex-debt` | Prioritized debt register |

Use the orchestrator for ambiguous requests. Invoke a specific skill when the intended stage is clear.
