# OhMyCodex v0.3.0

[English](v0.3.0.md) | 简体中文

v0.3.0 是破坏性版本。所有公开 Skill 都迁移到 `omc-*` 命名空间，不提供兼容别名。

## 迁移

| v0.3 之前 | v0.3 |
| --- | --- |
| `ohmycodex-orchestrator` | `omc-orchestrator` |
| `ohmycodex-init` | `omc-init` |
| `ohmycodex-discover` | `omc-discover` |
| `ohmycodex-spec` | `omc-spec` |
| `ohmycodex-architecture` | `omc-architecture` |
| `ohmycodex-implement` | `omc-implement` |
| `ohmycodex-qa` | `omc-qa` |
| `ohmycodex-debug` | `omc-debug` |
| `ohmycodex-refactor` | `omc-refactor` |
| `ohmycodex-review` | `omc-review` |
| `ohmycodex-release` | `omc-release` |
| `ohmycodex-debt` | `omc-debt` |
| `ohmycodex-team` | `omc-team` |

重新安装后，请更新已保存的提示词并新建任务。可移植调用方式是 `$omc-*` 或 `/skills`；`/omc-*` 解析只是客户端便利功能，不是自定义命令解析器。

## 新增能力

- `omc-doctor` 报告当前控制能力和安全降级路径。
- `omc-intentgate`、`omc-loop` 和 `omc-letgo` 增加由原生 Goal 支持的续跑能力，使用基于证据、可配置的阻塞阈值，不另设总循环次数限制。
- Scheduled 心跳仅用于外部等待，并在终态时清理。
- `omc-cn` 和 `omc-en` 以事务方式切换插件自有的英文或简体中文元数据。切换后需重启 Codex 或新建任务。
- Capability Resolver、Loop Ledger 和 Locale Manager 只使用 Python 标准库。

## 有意排除

此版本不添加自定义运行时、shell 循环、Stop Hook、守护进程、调度器、MCP Server、LSP Server、AST Server、App、模型供应商或遥测。Codex 原生的权限、信任、身份验证、Goal、Scheduled、MCP、发布、push、deploy、tag 和 publication 控制仍具有最高权威。
